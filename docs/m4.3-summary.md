## 0. TL;DR

- **前因**：M4.2 交付 5 个独立子系统（预算 / 观测 / 记忆 / 反思 / 事件），但缺 **横向串线**：① tick 内多表写入无统一 id；② NPC 间无直接交互路径。
- **交付**：M4.3 用 2 步纵切补齐这两条主干。
  - **M4.3.0**：tick 级 `trace_id`（uuid v4）贯穿 5 张表 + WS + `GET /api/engine/trace/:id` 运维接口
  - **M4.3.1**：`plan.speak.latest_say` → `scene_event{type='dialogue'}` 隐式自动化 + 回复链建模（`parent_event_id / conv_turn`）+ 硬上限回声保护 + 前端可视化
- **规模**：后端单测 152 → **180**（+28，超 roadmap 目标）；迁移 1 个；新 env 6 个；新 REST 1 个；前端零新依赖；bundle 增量远 <5 KB。
- **工时**：roadmap 估 2.75 天；实际 2 天（含 3 轮 live smoke + 1 轮闭环修复）。

## 2. 数据层改动汇总

### 2.1 扩列 7 个（1 个迁移文件）

```sql
ALTER TABLE npc_tick_log   ADD COLUMN trace_id CHAR(36) DEFAULT NULL;
ALTER TABLE ai_call_log    ADD COLUMN trace_id CHAR(36) DEFAULT NULL;
ALTER TABLE scene_event    ADD COLUMN trace_id CHAR(36) DEFAULT NULL;
ALTER TABLE npc_memory     ADD COLUMN trace_id CHAR(36) DEFAULT NULL;
ALTER TABLE npc_reflection ADD COLUMN trace_id CHAR(36) DEFAULT NULL;
ALTER TABLE scene_event    ADD COLUMN parent_event_id BIGINT DEFAULT NULL;
ALTER TABLE scene_event    ADD COLUMN conv_turn       INT    DEFAULT NULL;
```

### 2.2 索引 6 个

```sql
CREATE INDEX idx_npc_tick_log_trace   ON npc_tick_log(trace_id);
CREATE INDEX idx_ai_call_log_trace    ON ai_call_log(trace_id);
CREATE INDEX idx_scene_event_trace    ON scene_event(trace_id);
CREATE INDEX idx_npc_memory_trace     ON npc_memory(trace_id);
CREATE INDEX idx_npc_reflection_trace ON npc_reflection(trace_id);
CREATE INDEX idx_scene_event_parent   ON scene_event(parent_event_id);
```

**幂等性**：`migrate-m43.ts` 跑 2 次全部命中"列已存在 / 索引已存在，跳过"分支，无报错。

### 2.3 历史数据

不回填（保留 NULL）；查询层对 NULL 做 `COALESCE(trace_id, '<legacy>')` 即可，不影响 M4.2 行为。

## 3. 新增环境变量（6 个）

```env
# M4.3.0 trace 贯穿
TRACE_ID_ENABLED=true
TRACE_ID_LOG_FORMAT=short

# M4.3.1 多 agent 对话
DIALOGUE_AUTO_EVENT_ENABLED=true
DIALOGUE_CONTENT_MAX_LEN=200
DIALOGUE_ECHO_WINDOW_TICK=10
DIALOGUE_ECHO_MAX_TURN=3
```

**回退通道**：
- `TRACE_ID_ENABLED=false` → 5 张表 `trace_id` 全 NULL，WS / 接口降级，行为 = M4.2
- `DIALOGUE_AUTO_EVENT_ENABLED=false` → 不再自动注入 dialogue event；NPC 仍会说话但没有 `scene_event` 副本（回 M4.2 行为）
- `DIALOGUE_ECHO_MAX_TURN=0` → 禁用回声保护（放行所有，排障用）

## 4. 观测与运维

### 4.1 新增 REST（1 个）

```
GET /api/engine/trace/:trace_id
→ { counts: {...}, npc_tick_log: [...], ai_call_log: [...], scene_event: [...], npc_memory: [...], npc_reflection: [...] }
```

- 5 张表并发查询，单表失败降级为空不阻塞聚合
- scene_event 投影含 `parent_event_id / conv_turn`，可对账对话链
- uuid v4 格式校验 / 404 当全表 0 条

### 4.2 结构化日志

- `[dialogue.emit]` 失败 warn（不抛）
- `[intake.echo] 回声拦截 npc_id=<x> event_id=<y> actor=<name> conv_turn=<n> cap=<N>` —— 便于线上按关键字筛查回声触发

### 4.3 WS 事件

- 全部 `TickEvent` 带 `trace_id?`
- `scene.event.created` 带 `trace_id? / parent_event_id? / conv_turn?`（**本节点闭环修复**：之前 WS 少了 parent/conv，已补齐）

## 7. 已知限制 & 延后项

| # | 描述 | 级别 | 落地节点 |
|---|---|---|---|
| L-1 | 跨 tick 对话链受 `EVENT_LOOKBACK_SECONDS` 限制（默认 60s）；budget skip 会把 tick 间隔拉长致链断 | 中 | M4.4：建议默认调到 ≥120s，或引入 per-dialogue lookback |
| L-2 | 同 tick 内两 NPC 并发 intake，互看不到对方本 tick 产出；parent 需下一 tick 才挂上 | 低 | 设计使然；M4.4 turn-taking 调度可改 |
| L-3 | 手插事件 `conv_turn=null` 被选作 parent 时，子节点 `conv_turn` 从 `0+1=1` 重算，与其他链视觉上同 turn | 低 | 不影响核心逻辑；M4.4 统一兜底 |
| L-4 | 回声保护 `DIALOGUE_ECHO_WINDOW_TICK` 当前仅读 env 未真正消费（隐式窗口由 lookback + conv_turn 承担） | 低 | M4.4 若需精确按 tick 差拦截再启用 |
| L-5 | 前端 Sandbox 气泡 `💬 回应` 后缀以 ring buffer 为唯一数据源，ring 满 200 条后最老 parent 会查不到 | 低 | M4.4 前端重构窗口改 WeakMap 持久索引 |
| L-6 | 整节点 live smoke 受测试预算限制未做 4 NPC × 20 tick；实测 2 NPC × 5 tick + 手插链验证等价核心功能 | 低 | 未来 scale-out 时统一跑 |

**明确延后（与 roadmap §10 合并展开）**：对话 session 聚合表 / NPC↔NPC 线路图 / 玩家插话 / @mention 路由 / P95 观测面板 / Sandbox.vue 拆分 / 日程规划 —— 全部归入 M4.4 窗口。
