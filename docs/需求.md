# AINPC 需求文档

> 参考：行业 AI NPC 设计实践（大模型角色扮演、Prompt 支架、上下文管理等）

## 产品定位
- **游戏**，地图与交互为**核心玩法**

## 技术栈
- 前端: Vue 3 + Vite + TypeScript，地图渲染 **Phaser**
- 后端: Node.js
- 数据库: **MySQL**（持久化）+ **Redis**（实时状态，地图/NPC 位置/对话组）
- 实时推送: WebSocket（需鉴权，多用户隔离）

## 2. 角色 NPC 管理模块（需求设定）

### 2.1 核心概念

基于行业实践，AI NPC 需具备：
- **角色人格一致性**：通过 System Prompt 约束人设、口吻、行为边界
- **上下文连贯**：多轮对话需保持记忆与状态
- **模型能力适配**：不同场景可选高约束/低约束 Prompt 策略

### 2.2 功能需求

| 功能项 | 说明 | 优先级 |
|-------|------|--------|
| 角色 CRUD | 增删改查 NPC 基础信息（含名称、简介、背景、性格） | P0 |
| 角色背景 | 详细背景故事、出身、经历、关系等，支持长文本 | P0 |
| 角色性格 | 性格特质、待人方式、情绪倾向等 | P0 |
| 绑定 AI 配置 | 每个 NPC 关联一个 ai_config，决定调用的模型 | P0 |
| 系统提示词 | 自定义 System Prompt，可基于背景+性格自动组装 | P0 |
| 角色分类 | 按类型/场景分类（任务型、剧情型、客服型等） | P1 |
| 头像/立绘 | 支持头像 URL 或本地上传 | P1 |
| 对话管理 | 对话记录存储、历史回溯（可后续扩展） | P2 |

### 2.3 数据模型设计

```
npc 表
├── id              主键
├── name            角色名称
├── description     角色简介（简短概括，用于列表展示）
├── background      角色背景（详细背景故事、出身、经历、世界观等）
├── personality     角色性格（性格特质、待人方式、情绪倾向等）
├── avatar          头像 URL
├── ai_config_id    关联 ai_config，指定使用的 AI 模型
├── system_prompt   系统提示词（可基于 background+personality 自动组装或自定义）
├── category        分类：task/plot/custom 等（任务型/剧情型/自定义）
├── prompt_type     约束类型：high/low（高约束/低约束，影响生成风格）
├── status          状态：0 禁用 / 1 启用
├── sort            排序
├── created_at
└── updated_at
```

- **background（角色背景）**：详细背景设定，如出身、经历、与其它角色的关系、所处世界的规则等，用于 RAG 检索或注入 System Prompt
- **personality（角色性格）**：性格描述，如开朗/冷淡、谨慎/冲动、善良/狡黠等，以及待人接物风格
- **system_prompt**：可由 background + personality 自动拼接，或完全自定义，示例结构：
  ```
  你是{name}，{description}。
  背景：{background}。
  性格特点：{personality}。
  说话风格：{tone}。
  禁止事项：{forbidden}。
  ```

- **prompt_type**：高约束(HCP)适合任务发布型，低约束(LCP)适合需即兴发挥的角色

### 2.4 与 AI 配置模块的关系

```
ai_config (已有)     npc (新增)
     │                    │
     └── ai_config_id ────┘
         每个 NPC 必须绑定一个启用的 AI 配置
```

### 2.5 迭代范围说明

| 本期实现 | 后期迭代 |
|---------|---------|
| NPC 配置管理（CRUD、背景、性格、AI 绑定、系统提示词） | 用户-NPC 对话 API |
| 角色分类、头像、基础枚举 | 对话记录存储（npc_conversation 表） |
| - | 记忆/历史回溯、上下文管理 |
| - | category、prompt_type 枚举值产品定稿 |

**当前优先实现**：NPC 配置的增删改查及基础字段管理，对话、记忆等功能留待后期迭代。

## 5. NPC 之间聊天模块（规划）

> 参考：AI-Town handleAgentInteraction、divideIntoGroups

| 功能项 | 说明 | 优先级 |
|-------|------|--------|
| 1v1 对话 | 两 NPC 之间自动对话，converse + rememberConversation | P1 |
| 多人对话组 | 按距离分组，decideWhoSpeaksNext 决定发言者 | P2 |
| 关系记忆 | memories 表 relationship 类型，存储 NPC 间关系 | P1 |
| 对话总结 | 每轮对话结束 LLM 总结入库 | P1 |
| 反思机制 | importanceScore 达阈值时 reflectOnMemories | P2 |

### 5.1 核心流程（AI-Town 参考）

```
tick → divideIntoGroups
  ├── 对话组：handleAgentInteraction
  │   ├── decideWhoSpeaksNext（LLM 决定下一个发言者）
  │   ├── walkAway（LLM 决定是否离开）
  │   ├── startConversation（打招呼 + 关系记忆）
  │   ├── converse（身份 + 关系 + 2 条记忆）
  │   └── rememberConversation（总结入库）
  └── 个人组：reflectOnMemories（重要性 > 500 时提炼 insight）
```

### 5.2 成本优化

- 严格控制注入记忆条数（2–5 条），避免 token 爆炸
- 对 memory 做重要性筛选与摘要压缩后再注入

详见：`交互.md` 第三、四、五节。
