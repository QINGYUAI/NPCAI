# 地图模块架构梳理

> 梳理时间：2026-03-18 | 重构时间：2026-03-18

## 一、模块概览

地图模块负责：**地图 CRUD**、**物品驱动格子弹窗**、**NPC 绑定与场景**、**AI 生成/图片转地图**、**Redis 实时状态**、**WebSocket 推送**。

**重构后技术栈**：Zod 校验、Controller 拆分、Service 层、Composables（Vue 3）。

---

## 二、数据模型

### 2.1 核心表

| 表名 | 说明 | 迁移脚本 |
|------|------|----------|
| `game_map` | 地图主表：name, width, height, tile_data(JSON), metadata | `db:migrate-map` |
| `npc_map_binding` | NPC 与地图绑定及初始位置 (npc_id, map_id, init_x, init_y) | `db:migrate-map` |
| `item` | 物品定义：name, footprint(JSON), tile_value, metadata | `db:migrate-item` |
| `item_map_binding` | 物品与地图关联：item_id, map_id, pos_x, pos_y, rotation | `db:migrate-item` |
| `npc_map_item` | NPC 与地图物品关联（nearby 等） | `db:migrate-item` |
| `npc_npc_conversation` | NPC 间对话会话（持久化） | `db:migrate-map` |
| `npc_npc_message` | NPC 间对话消息 | `db:migrate-map` |

### 2.2 数据约定

- **tile_data**：二维数组，`0` = 可行走，非 `0` = 障碍（值对应 tile_types）
- **footprint**：物品的 2D 形状，`1` = 障碍，`0` = 门洞/通道
- **metadata**：含 `tile_types: Record<number, { name, color }>` 用于渲染

### 3.2 控制器职责拆分（map.ts）

| 功能域 | 函数 | 说明 |
|--------|------|------|
| **地图 CRUD** | `getMapList`, `getMapById`, `createMap`, `updateMap`, `deleteMap` | 基础增删改查 |
| **AI 生成** | `generateMapContent`, `convertLayoutImageToMap` | 文本/图片 → 地图 JSON |
| **物品管理** | `getMapItems`, `addMapItem`, `removeMapItem` | 物品放置与移除 |
| **NPC 绑定** | `getMapBindings`, `addMapBinding`, `removeMapBinding` | NPC 与地图绑定 |
| **场景控制** | `initScene`, `startMap`, `pauseMapController`, `resumeMapController` | Redis 初始化与 wander 启停 |
| **状态查询** | `getSceneState` | 实时 NPC 位置与 running 状态 |

### 3.3 服务层职责

| 服务 | 职责 |
|------|------|
| **mapScene** | 从 Redis 构建 `SceneStateData`（npcs + running），Redis 空时从 `npc_map_binding` 兜底 |
| **itemMap** | `deriveTileDataFromItems`、`deriveTileDataFromBindings`、`findOrCreateItem`、`getTileTypesForMap`、`getNearbyItemNames` |
| **wander** | 定时 tick、AI 决策移动、相遇对话、`npc_map_item` 写入、`broadcastToMap` 推送 |

| 方法 | 路径 | 说明 |

### 3.5 Redis 结构

| Key 模式 | 说明 |
|----------|------|
| `map:{mapId}:npcs` | Set，该地图上的 NPC id 列表 |
| `map:{mapId}:npc:{npcId}` | Hash：x, y, state, groupId, updatedAt |
| `map:{mapId}:paused` | 1 = 已暂停 |

## 五、流程梳理

### 5.1 创建地图（两种模式）

1. **物品驱动**：传入 `items` → 后端 `findOrCreateItem` + `item_map_binding` → `deriveTileDataFromItems` → 写入 `game_map.tile_data`
2. **直接传入**：传入 `tile_data` + `metadata` → 直接写入 `game_map`

### 5.2 AI 生成地图

1. 文本：`generateMapContent` → LLM 返回 JSON → `parseMapGenerateJson` → 前端预览后可 `createMap`
2. 图片：`convertLayoutImageToMap` → Vision 模型识别 → 同上

### 5.3 场景运行流程

```
1. 用户点击「开始」→ startMap
2. 若 Redis 无数据 → initSceneData：从 npc_map_binding 读取，随机分配可行走格子，写入 Redis
3. resumeMap(mapId) → wander 开始 tick
4. 每个 tick：AI 决策移动 → 更新 Redis → broadcastToMap
5. 前端：WebSocket 或轮询 getSceneState → 更新 Phaser 显示
```
